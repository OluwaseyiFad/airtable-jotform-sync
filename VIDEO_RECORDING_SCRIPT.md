### Before You Start Recording

#### Equipment & Software

- [ ] **Screen Recording Software**
  - Options: OBS Studio (free), Camtasia, ScreenFlow
  - Set to record at 1920x1080 (1080p) minimum
  
- [ ] **Microphone**
  - Use external microphone if possible
  - Test audio levels before recording
  - Eliminate background noise

- [ ] **Browser Setup**
  - Use Chrome or Firefox in a clean profile
  - Disable extensions that might show notifications
  - Close unnecessary tabs
  - Set browser zoom to 100%

- [ ] **Preparation**
  - Have test Jotform form ready
  - Have test Airtable base ready
  - Practice each segment before recording
  - Prepare cursor highlighter (optional but recommended)

#### Recording Settings

- [ ] **Resolution:** 1920x1080 (1080p) or higher
- [ ] **Frame Rate:** 30 fps minimum
- [ ] **Audio:** 44.1 kHz or 48 kHz, stereo
- [ ] **Cursor:** Highlight clicks (yellow circle effect)
- [ ] **Zoom:** Use for detailed actions (e.g., clicking small buttons)

#### Post-Production Tools

- [ ] Video editor: DaVinci Resolve (free), Adobe Premiere, iMovie
- [ ] Thumbnail creator: Canva, Photoshop
- [ ] Caption/subtitle software: Auto-generated + manual review

## üí° Video Tips & Best Practices

### Recording Environment

**‚úÖ DO:**
- Record in a quiet room
- Use consistent lighting
- Close unnecessary applications
- Disable notifications
- Use a pop filter for microphone

**‚ùå DON'T:**
- Record with background noise (AC, traffic, pets)
- Use built-in laptop mic if avoidable
- Have cluttered desktop visible
- Use auto-focus cameras (creates distracting refocus)

### Presentation Style

**‚úÖ DO:**
- Speak clearly and at moderate pace
- Use a friendly, conversational tone
- Pause between sections
- Repeat key points
- Use analogies and examples
- Smile (comes through in voice)

**‚ùå DON'T:**
- Rush through material
- Use jargon without explanation
- Assume prior knowledge
- Apologize for mistakes (just re-record)
- Read word-for-word from script

### Screen Recording

**‚úÖ DO:**
- Record at 1920x1080 or higher
- Use cursor highlighting
- Close unnecessary tabs/windows
- Zoom in on small UI elements
- Keep mouse movements smooth and deliberate
- Pause between actions

**‚ùå DON'T:**
- Record at low resolution
- Move mouse erratically
- Click too fast
- Show personal information
- Include browser bookmarks with personal info

### Accessibility

**‚úÖ DO:**
- Add captions/subtitles
- Use high contrast colors
- Make text large enough to read
- Describe what you're doing verbally
- Provide transcripts

**‚ùå DON'T:**
- Rely solely on color to convey meaning
- Use flashing effects (seizure risk)
- Use tiny fonts
- Assume everyone can see details

## üìä Video Analytics & Improvement

### Metrics to Track

After publishing, monitor:

- **View count** - Are people watching?
- **Watch time** - How long do they watch?
- **Drop-off points** - Where do they stop watching?
- **Likes/dislikes** - Are people finding it helpful?

### Continuous Improvement

Based on analytics and feedback:

1. **If people drop off early:**
   - Intro might be too long
   - Get to the point faster
   - Add more visual interest

2. **If people drop off mid-video:**
   - Pace might be too slow
   - Content might be too basic/advanced
   - Consider splitting into shorter videos

3. **If comments show confusion:**
   - Create supplementary videos
   - Add more detailed explanations
   - Update documentation

4. **If engagement is low:**
   - Improve thumbnails
   - Better video titles
   - Add more personality/energy
